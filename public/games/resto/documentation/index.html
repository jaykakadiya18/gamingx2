A) Introduction - top

Open Restaurant is a HTML5 game where you have to run a restaurant and bring your management skills to every diner! Survive the daily target to get highscore.

The ZIP package contains the game with 1280×768 resolution that scales to fit the whole screen device, but it may not be perfectly full screen.

How To Play:

B) Getting Started - top

To install the game just upload folder 'game' to your server. The game won't run locally with some browser like Chrome due to some security mode.

You need a website that runs PHP to make facebook share button work, and make sure to change Facebook Open Graph meta and Twitter meta in index.html, just replace [GAME_URL] to your game URL.

        <!-- for Facebook -->
        <meta property="og:image" content="[GAME_URL]/share.jpg" />
        <meta property="og:url" content="[GAME_URL]" />
        
        <!-- for Twitter -->
        <meta name="twitter:image" content="[GAME_URL]/share.jpg" />

position_arr array place all the objects in isometric grid, below is the explanation of each most important position_arr array item:

  1. id - id name from objects_arr
  2. r - isometric row position
  3. c - isometric column position
  4. slot - an array list of row and column position for depth masking
  5. block - an array list of row and column position for non-walkable
  6. type - type of objects name for interation
  7. table - link to table, value should be array number (only work when type is 'chair')
  8. exact - an array list of row and column for exact positon when interact, instead of go nearby

D) CSS Files and Structure - top

I'm using two CSS files in this game. The first one is a generic reset file. Many browser interpret the default behavior of html elements differently. By using a general reset CSS file, we can work round this. This file also contains some general styling, such as anchor tag colors, font-sizes, etc. Keep in mind, that these values might be overridden somewhere else in the file.

The second file contains all of the specific stylings for the page.

E) JavaScript - top

This game using Javascript files below.

  1. jQuery is a cross-platform JavaScript library designed to simplify the client-side scripting of HTML.

  2. Detect Mobile Browser is a open source scripts to detect mobile browsers and phones.

  3. CreateJs plugin is a suite of modular libraries and tools which work together to create interactive content on open web technologies via HTML5.

  4. TweenMax is an extremely fast, lightweight, and flexible animation tool that serves as the foundation of the GreenSock Animation Platform (GSAP).

  5. PathFinding.js is a comprehensive path-finding library in javascript.

  6. The game have the following js files
    • init.js : check if browser or device support

    • loader.js : loader to load all game images

    • main.js : initiate game setup and browser resize function

    • mobile.js : mobile orientation change

    • canvas.js : canvas setup and resize

    • sound.js : sound event

    • game.js : game play and logics

    • plugins.js : additonal useful plugins

Complete game flow:

  1. The index.html file start init.js for browser detection
  2. If browser is supported, init loader.js to start load asserts with loading progress
  3. For mobile the rotate instruction shows when device is in portrait view, detect by mobile.js
  4. When all asserts contained in "/assets" folder are loaded, the game start construct canvas.js and main.js from game.js thats shows game menu
  5. If user click begin button in game menu, the game will start with game.js
  6. If user tap on the menu counter when customer in queue line, pop message appear with number for table
  7. If user tap on the empty table, manager will bring customers to the table
  8. If user tap on the table with food order pop message, manager will take food order
  9. If user tap on the kitchen with food deliver pop message, manager will take food and immediate display indicator on which table should the food serve to
  10. While the manager is hand with food, no interaction with other objects until the food is serve
  11. If user tap on the table with payment pop message, manager accepet payment and customers will leave, at the same time game will score
  12. If user tap on the table with customers left, manager will clean the table
  13. If user hti the score and time's up, a new day will start with new score target
  14. If user didn't hit the daily target when time is up, the game will end
  15. If user click replay button at game result, it will restart game

G) Isometric Map - top

You can find isometric settings in game.js: 

var isometricData = {guide:false, x:256, y:353, row:20, column:7, width:32.3, height:19, xCorrect:29, yCorrect:16.8, array:[]};

Below is the explanation of each most important isometricData item:

  1. guide - true/false to enable isometric grid
  2. x - position x
  3. y - position y
  4. row - total row
  5. column - total column
  6. width - isometric width
  7. height - isometric height
  8. xCorrect - xCorrect
  9. yCorrect - yCorrect
  10. array - array list to store isometric grid
